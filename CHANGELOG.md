# 更新记录

## v0.4.1 - 2026-07-18

本版本汇总提交 `709d188`（v0.4.0）之后至 `35c314e` 的 29 次提交。核心变化是将站点从旧版数据驱动静态页面迁移为以 Markdown 为唯一内容源的 VuePress 知识库，并重新收敛表征学习的内容结构。

### 站点架构

- 建立 VuePress 2、Vite、Vue 3 与 Theme Hope 站点骨架。
- 将可编辑内容统一迁入 `docs/`，按表征学习、深度学习项目、深度学习研究和文献库四个板块组织。
- 使用独立的顶栏与侧栏配置生成导航，支持 Markdown、数学公式、目录和页面元数据渲染。
- 完成旧版 HTML、CSS、原生 JavaScript、KaTeX 和 marked 运行时的退役，删除重复内容源与旧构建脚本。

### 编辑与质量检查

- 引入 pnpm 工程配置、Prettier 和 Markdownlint。
- 配置 Husky 与 nano-staged，在提交前自动格式化暂存文件并检查 Markdown。
- 补充本地开发、构建、清缓存、内容添加和部署说明。

### 部署

- 建立 GitHub Actions 测试与 GitHub Pages 部署流程。
- 保留 Vercel 自定义域名部署，形成 GitHub Pages 与 `rea.thebear617.cn` 双入口。
- 修正基础路径、板块导航与本地开发体验，完成新站线上切换。

### 文献库

- 将文献索引、学术会议与期刊内容迁移为本地 Markdown。
- 集成《Attention Is All You Need》中文全文及论文图片。
- 删除英文全文副本，让文献索引中的论文标题直接指向中文全文。

### 表征学习

- 将原有卡片和多文件内容逐步合并为一个基础知识 Markdown 页面。
- 保留“表征学习的数学模型和研究目的”与“线性表征跟非线性表征的比较”。
- 建立“表征约束 × 训练目标构造来源”的双轴框架。
- 将典型方法整理为双轴坐标示例，区分主要信号来源与辅助来源。
- 将常见自监督方法的实现机制作为基础知识保留，不再设置为独立分类维度。
- 将板块入口收敛为“学习路线”，通过锚点给出统一阅读顺序。

### 内容迁移

- 完整迁移深度学习项目的八阶段实践路线。
- 完整迁移深度学习研究的问题、表征、架构、训练、调参与工程化内容。
- 保留迁移后的中文目录、页面顺序和板块边界，站点内容不再依赖旧 JavaScript 数据文件。
